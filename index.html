TAO-C2 PLUS TWO (V 1.6.2)

TAO-C2 is a C++ version of the Windows TAO-C2 python framework designed for educational research into endpoint management, API-based command-and-control (C2) architectures, and process monitoring techniques.

Key Features

  • Hybrid C2 Architecture: Demonstrates dual-mode communication using standard web APIs (JSONBin) or DNS Tunneling (dnslib) with fall-back mechanisms.
  • System Integrity & Persistence: Done through automated Windows Task Scheduling (schtasks) and file attribute manipulation.
  • Process Hooking: A C++ Stub Loader (ldr.cpp) and hook.dll (hook.cpp) to evade system monitoring tools.

Modules

Currently, this version only offers core main modules cvm.dll and prep.dll. I am currently working on re-integrating the python modules.

  • As per discussed before, cvm.dll is the Sandbox awareness module and prep.dll is environment preparation and obfuscation engine.

Setup and Compilation

1. Requirements

  • Controller: Python 3.x with tkinter, requests, dnslib, and cryptography.
  • Agent: MinGW64 compiler environment or Microsoft Visual Studio

2. Configuration

  • Configure your primary HTTP communication backend (JSONBin credentials) within the config_key.json file.
  • The Python controller (remote_c.py) provides an (TBA) tool interface for manual credential entry and agent management.

3. Compilation

Using MinGW64:

  • Place the source files in the compiler directory.
  • Run Compile.bat as Administrator (required for setting development-time security exclusions).
  • The script will compile the core agent (htr.exe), the modules, and the stub loader (ldr.exe).

Note: The script is configured to clean up source objects and compiler tools upon successful build to simulate a deployment-ready environment.

Using Microsoft VS:

  • Additional dependencies: ws2_32.lib;wininet.lib;crypt32.lib;advapi32.lib;shell32.lib;user32.lib;
  • Change Runtime Library to Multi-threaded (/MT)
  • Set SubSystem to Windows (/SUBSYSTEM:WINDOWS) or comment out the void InitDebugConsole and its associated functions.
  • Set Build Configuration to Release and x64

T.A.O-RAT Python Framework

A modular windows-based Remote Control & Administration Tool written in Python and C++. It is used for educational research into endpoint management, API-based command-and-control (C2) architectures, and process monitoring.

Key Features

  • Remote Command Execution: Demonstrates reverse shell architecture via the def cmd_(cmd) function.
  • System Integrity: Modules to test environment awareness (VM detection) and security configuration (Defender interaction).
  • Persistence Modeling: Implementation of automated task scheduling for persistent administration.
  • Process Hooking: A C++ Stub Loader (ldr.cpp) and hook.dll (hook.cpp) to see how User-Mode hooking can influence system monitoring tools like Task Manager.

Modules

Modules are accessible via the "Tools" panel which includes:

  • Surveillance modules - Including screen capture and input logging for auditing user activity.
  • File Exfiltration module - File exfiltration testing using the GoFile.io API.
  • XMRIG module - Deployment of distributed computing tasks (XMRIG integration).

Setup

  • Install third-party python modules: pip install requests pyautogui cryptography pynput psutil
  • Configuration: Set up a JSONBin.io account and put your credentials in config_key.json (for the agent), and enter manually in the remote app.
  • Compile: ldr.cpp and hook.cpp by running the Compile.bat file found in the Compile folder. Then place the compiled files (exe and dll) inside the modules folder.
  • Compile 2: The htr.py to an executable via the Compile.bat file found in the payload folder. (You must have PyInstaller installed).
  • Place: The modules folder inside the directory of the compiled htr.exe. You must have the config_key.json with your credentials there.
  • Run: Run htr.exe and then open your htr_remote script to start.
