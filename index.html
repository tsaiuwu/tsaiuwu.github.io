DLL Hollowing with EDR Evasion

Overview

This project demonstrates how attackers attempt to bypass modern Windows defenses such as:

  • ETW (Event Tracing for Windows) telemetry
  • AMSI (Antimalware Scan Interface) scanning
  • EDR hooks in ntdll.dll and kernel32.dll

It implements:

  • ETW patching (EtwEventWrite stubbed out)
  • AMSI patching (AmsiScanBuffer forced to return benign results)
  • Full-spectrum unhooking of system DLLs by restoring clean .text sections
  • Module stomping: loading a payload from resources and overwriting a benign DLL (winmm.dll) in memory
  • Thread suspension/resumption to safely patch memory

Goals

  • Show how attackers neutralize telemetry and scanning.
  • Provide defenders with insight into how unhooking and module stomping work.
  • Help blue teams build detections and mitigations against these techniques.

Defensive Takeaways

  • Monitor for unexpected memory patches in ntdll.dll and kernel32.dll.
  • Detect suspicious calls to VirtualProtect/WriteProcessMemory targeting security functions.
  • Watch for module stomping behavior (legitimate DLLs overwritten in memory).
  • AMSI bypasses often rely on short instruction patchesâ€”signature detection is possible.

Usage and Testing

Do not run this on production systems. For testing:

  1. Compile as a DLL with embedded payload on RCDATA renamed as 101.
  2. Inject into a test process in a sandboxed lab environment.
  3. Observe how ETW/AMSI events disappear and how payload execution occurs.

This repository contains proof-of-concept code for studying advanced malware evasion techniques on Windows. It is intended solely for educational and research purposes in controlled environments. Do not use this code for unauthorized activity. Misuse may violate laws and cause harm.

References

MITRE ATT&CK: Defense Evasion, AMSI bypass research papers, ETW patching techniques in malware analysis blogs.

TAO-C2 PLUS TWO (V 1.6.2)

TAO-C2 is a C++ version of the Windows TAO-C2 python framework designed for educational research into endpoint management, API-based command-and-control (C2) architectures, and process monitoring techniques.

Key Features

  • Hybrid C2 Architecture: Demonstrates dual-mode communication using standard web APIs (JSONBin) or DNS Tunneling (dnslib) with fall-back mechanisms.
  • System Integrity & Persistence: Done through automated Windows Task Scheduling (schtasks) and file attribute manipulation.
  • Process Hooking: A C++ Stub Loader (ldr.cpp) and hook.dll (hook.cpp) to evade system monitoring tools.

Modules

Currently, this version only offers core main modules cvm.dll and prep.dll. I am currently working on re-integrating the python modules.

  • As per discussed before, cvm.dll is the Sandbox awareness module and prep.dll is environment preparation and obfuscation engine.

Setup and Compilation

1. Requirements

  • Controller: Python 3.x with tkinter, requests, dnslib, and cryptography.
  • Agent: MinGW64 compiler environment or Microsoft Visual Studio

2. Configuration

  • Configure your primary HTTP communication backend (JSONBin credentials) within the config_key.json file.
  • The Python controller (remote_c.py) provides an (TBA) tool interface for manual credential entry and agent management.

3. Compilation

Using MinGW64:

  • Place the source files in the compiler directory.
  • Run Compile.bat as Administrator (required for setting development-time security exclusions).
  • The script will compile the core agent (htr.exe), the modules, and the stub loader (ldr.exe).

Using Microsoft VS:

  • Additional dependencies: ws2_32.lib;wininet.lib;crypt32.lib;advapi32.lib;shell32.lib;user32.lib;
  • Change Runtime Library to Multi-threaded (/MT)
  • Set SubSystem to Windows (/SUBSYSTEM:WINDOWS)

T.A.O-RAT Python Framework

A modular windows-based Remote Control & Administration Tool written in Python and C++. It is used for educational research into endpoint management and C2 architectures.

Key Features

  • Remote Command Execution: Demonstrates reverse shell architecture via the def cmd_(cmd) function.
  • System Integrity: Modules to test environment awareness (VM detection) and security configuration.
  • Persistence Modeling: Implementation of automated task scheduling for persistent administration.

Setup

  • Install third-party python modules: pip install requests pyautogui cryptography pynput psutil
  • Configuration: Set up a JSONBin.io account and put your credentials in config_key.json.
  • Compile: ldr.cpp and hook.cpp by running the Compile.bat file found in the Compile folder.
